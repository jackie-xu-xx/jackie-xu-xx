# 你的 OpenClaw 为什么越用越蠢？

之前我们聊过怎么优化 memory、压缩 context、用 code hook 规范行为。这些都有用，但跑了五周下来，我发现还漏了一个东西——**Workspace 结构**。

回想一下你的 bot 是不是也这样：同一个问题 debug 了 20 次，第二天照犯？规则强调了 N 遍，它还是记不住？

Bot 每次醒来都是一张白纸。它要从你的文件里搞清楚：我是谁？该干嘛？工具在哪？如果 workspace 乱七八糟，它光"找路"就烧掉一半 context，哪还有余量干正事？

下面是我这几个月摸索出来的结构，直接抄就行。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

## 我的 workspace 长这样

```asciidoc
workspace/
├── AGENTS.md      ← 启动入口
├── SOUL.md        ← 人格定义
├── USER.md        ← 关于你
├── MEMORY.md      ← 长期记忆
├── TOOLS.md       ← 工具笔记
├── HEARTBEAT.md   ← 定时任务
├── TODO.md        ← 待办
│
├── memory/
│   ├── 2026-02-20.md   ← 每日 log
│   ├── insights/       ← 压缩洞察
│   └── lessons/        ← 踩坑记录
│
└── ~/clawd/skills/     ← 所有 skills
```

## 每个文件干嘛

- **AGENTS.md** — Bot 醒来第一件事读这个。写清楚有哪些文件、什么任务读什么 skill、红线在哪。注意：这是目录，不是规则手册。

- **SOUL.md** — 人格和语气。我写了"别说废话"、"有观点直说"、"可以吐槽"、"骂人可以但要有道理"。还可以写偏好：比如"代码注释用英文"、"回复用中文除非对方先用英文"。没这个，bot 就是个客服机器人。

- **MEMORY.md** — 长期记忆，只放精华。流水账放 memory/ 目录。

- **memory/** — 分层存储：原始日志 → 压缩洞察 → 踩坑教训。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

## 三个坑，我都踩过

### 1. Skills 到处乱放

一开始 skills 装到 `/opt/clawdbot/skills/`，后来又在 workspace 里建了几个。结果 bot 改了 A 版本，跑的是 B 版本，debug 两小时才发现。

**现在的规则：统一放 `~/clawd/skills/`，零歧义。**

### 2. 一个功能三个文件夹

有段时间同时存在 x-poster/、twitter-bot/、x-tools/，都是发推相关的。让 bot 改配置，它改了 A，实际跑的是 B。

**教训：一个功能一个 skill，一个 skill 一个文件夹。**

### 3. 规则写了但 bot 不听

AGENTS.md 写了一堆"必须这样做"，bot 隔几天就忘。后来才明白：context 压缩时规则被丢了。它不是不听，是真没看到。

**解法：规则写进输出格式。**

比如我强制所有外发内容带 `Humanizer: ✅` 标记——没这行就是没过检查，我一眼就能看出来。再比如修改 AGENTS.md 前必须输出"三问自检"结果，没有这段就不允许提交。

格式约束比写在文档里可靠，因为它不依赖 bot "记住"规则。

## 几个实战技巧

### AGENTS.md 的三问测试

往里加东西之前，先问：

- 删掉会影响 99% 日常操作吗？不会就别放
- 以后需要扩展吗？会就独立成 skill，这里只放指针
- 新人 5 分钟能理解吗？不能就太复杂了

### 定期压缩记忆

每日 log 会爆炸。我写了个脚本，每天把前一天的 log 压缩 10:1：保留决策和结论，删掉过程细节，核心存 insights/。

### 给记忆加 TTL

不是所有东西都值得永久保存：

- [P0] — 永不过期
- [P1] — 90天
- [P2] — 30天

### 维护 Task Checklist

AGENTS.md 里放一张表：

- 写文章/配图 -> contentgen-skill
- 发 X/回复 -> x-engagement-skill
- 高风险操作 -> roundtable-skill

Bot 收到任务先查表，不靠脑子记

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

## 总结

**四条原则：**

1. **入口清晰** — AGENTS.md 是目录
2. **职责分离** — 每个文件干一件事
3. **记忆分层** — 原始→压缩→精华
4. **规则落地** — 写进格式，不靠 context

这套结构跑了五周，bot 基本不掉链子了。
